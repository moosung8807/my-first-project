# AGENTS.md

## 프로젝트 개요
- 이 저장소는 개인 투자자를 위한 한국어 포트폴리오 리밸런싱 웹사이트이며, 정적 사이트 구조를 기본으로 사용합니다.
- 메인 진입점은 `index.html`입니다.
- 프론트엔드 코드는 `assets/css`, `assets/js` 아래에 있습니다.
- 서버 측 코드는 `functions/api` 아래에 있으며, `/api/*` 경로를 제공하는 서버 환경에서 동작하는 것을 전제로 합니다.
- 데이터 생성 및 검증 유틸리티는 `scripts/`, `data/` 아래에 있습니다.

## 작업 방식
- 명시적인 요청이 없는 한 현재 구조를 유지합니다. 즉, 프레임워크를 도입하지 말고 HTML, CSS, 브라우저 JavaScript 기반 구조를 유지합니다.
- 파일 구성과 네이밍 스타일에 맞는 작고 목적이 분명한 수정부터 우선합니다.
- 별도 요청이 없는 한 사용자에게 보이는 문구는 한국어로 유지합니다.
- 작업 목적이 해당 영역 변경이 아닌 경우 기존 SEO, 메타데이터, 구조화 데이터, Open Graph 태그는 보존합니다.

## 프론트엔드 규칙
- 페이지 구조와 스크립트가 참조하는 DOM id/class의 기준 파일은 `index.html`로 간주합니다.
- 셀렉터나 마크업의 id/class를 바꾸기 전에는 반드시 `assets/js`, `assets/css`에서 연결된 동작을 함께 확인합니다.
- 로직을 분리할 수 있다면 `assets/js/app.js`에 큰 덩어리로 추가하기보다 기존 helper 모듈을 확장하는 방식을 우선합니다.
- 데스크톱과 모바일 모두에서 반응형 동작이 유지되도록 수정합니다.
- 요청이 기능 수정이나 국소 수정이라면 불필요한 시각적 전면 개편은 피합니다.

## 서버리스 규칙
- `functions/api/quote.js`는 현재의 서버리스 형태인 `export async function onRequestGet(context)`와 호환되게 유지합니다.
- 명시적 요청이 없는 한 비밀값을 노출하거나 특정 배포 플랫폼 전용 설정을 추가하지 않습니다.
- 현재가 조회 프록시의 입력 검증과 외부 응답 오류 처리는 보수적으로 유지합니다.

## 데이터 및 스크립트 규칙
- ETF 검증이나 OG 자산 생성 작업은 `scripts/`의 기존 Node 스크립트를 우선 활용하고, 같은 흐름을 다른 곳에 다시 구현하지 않습니다.
- `data/` 아래 파일은 신중히 다루고, 가능하면 생성 결과물 JSON은 스크립트를 통해 갱신합니다.
- 작업 목적이 해당 파일 수정이 아닌 한 `data/`, `assets/og/` 아래 바이너리 자산은 수정하지 않습니다.

## 검증
- 기본적으로 작업 후 자동 검증은 수행하지 않습니다.
- 검증은 사용자가 명시적으로 요청한 경우에만 수행합니다.
- 검증을 요청받았다면 관련 범위에 맞는 가장 작은 단위의 검증부터 우선합니다.
- 오래 걸리거나 rate limit 위험이 큰 검증은 꼭 필요할 때만 실행합니다.
- 요청받은 검증을 로컬에서 수행할 수 없는 경우에는 최종 응답에서 명확히 밝힙니다.

## 변경 안전성
- 파괴적인 가정을 하기 전에 `git status`를 확인합니다. 작업 트리에 사용자가 만든 파일이 있을 수 있습니다.
- 관련 없는 untracked 파일을 삭제하거나 덮어쓰지 않습니다.
- 새 서드파티 의존성, 빌드 도구, 배포 전용 인프라를 추가해야 한다면 먼저 확인을 구합니다.
